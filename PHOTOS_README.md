# Добавление фотографий объектов

Чтобы использовать свои фотографии вместо placeholder-изображений с Unsplash, положите файлы в папку `frontend/public/photos/items/`.

## Быстрый старт

1. Откройте папку `frontend/public/photos/items/`
2. Скопируйте туда свои фото
3. Перезапустите фронтенд (`npm run dev`) или пересоберите (`npm run build`)

## Соглашение об именах

| Сценарий | Имя файла | Пример |
|----------|-----------|--------|
| Одно фото на объект | `{id}.jpg` | `1.jpg` для объекта с id=1 |
| Галерея (несколько фото) | `{id}_1.jpg`, `{id}_2.jpg` … | `1_1.jpg`, `1_2.jpg`, `1_3.jpg` |

ID объекта можно узнать в админке или из ответа API `/api/items`.

## Форматы и рекомендации

- **Форматы:** `.jpg`, `.jpeg`, `.png`, `.webp`
- **Размер:** 800×600 px или крупнее
- **Вес:** до 300 КБ на фото (для быстрой загрузки в Mini App)
- **Соотношение сторон:** 4:3 для галереи, 1:1 для аватаров в stories

## Пример структуры

```
frontend/public/photos/items/
├── 1_1.jpg       # Домик №1 — главное фото
├── 1_2.jpg       # Домик №1 — интерьер
├── 1_3.jpg       # Домик №1 — вид снаружи
├── 9_1.jpg       # Баня на берегу
├── 9_2.jpg       # Баня — интерьер
├── 10_1.jpg      # Купель
└── ...
```

Если для объекта не найдено ни одного фото, приложение покажет красивые placeholder-фото с Unsplash.

## Фото общих зон

Фото территории, реки, леса можно положить в `frontend/public/photos/base/` и использовать по прямой ссылке `/photos/base/имя_файла.jpg`.
